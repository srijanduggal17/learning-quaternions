- (x,y,z) are just like a 3D vector

vector is context-dependent, just like
a 3D vector doesn't convey the reference
frame it is in ^y9I4qZ0G

4D space - (x, y, z, w)

w + x*i + y*j + z*k ^IUX6qWmu

cos( theta/2 ) + (a*I + b*j + c*k)(sin (theta/2 ) ^vSav65cr

Real part: w -> corresponds to the angle of rotation
Imaginary part: (x, y, z) -> corresponds to the axis of rotation ^KzfSUssM

Note that the frame which the axis of rotation is defined in is unclear
from the quaternion itself, but it depends on the context

Unit quaternion (x^2 + y^2 + z^2 + w^2 = 1)

Axis is also a unit vector (a^2 + b^2 + c^2 = 1) ^umVLXIWR

Can represent orientations ^cwnZ37G2

Same axis/angle situation as when used to represent rotations

default reference frame to get to the basis vectors of your orientation.
